# Story 1.6: Proc-Glow Border Pulse

**Status:** Done
**Epic:** 1 — Core Display
**Version:** v0.0.6
**Commits:** `bb0a1c7`

## Context

Some spells have a "proc" state: a buff activates that makes the spell free, instant, or significantly empowered. Blizzard signals this via the spell activation overlay glow on the action bar. The addon needed to replicate this visual feedback — a gold border pulse — so the player knows the current suggestion is also a proc opportunity.

## Acceptance Criteria

- [x] When the suggested spell has an active proc glow, the icon's backdrop border pulses gold
- [x] When the proc glow ends, the border returns to its default color
- [x] The pulse animates (gray → gold transition) via a ticker, not an instant color change
- [x] Proc state is detected via WoW events, not per-frame polling

## Implementation Notes

- Listens to `SPELL_ACTIVATION_OVERLAY_GLOW_SHOW` and `SPELL_ACTIVATION_OVERLAY_GLOW_HIDE` events
- `C_SpellActivationOverlay.IsSpellOverlayed(spellID)` is also checked during `Refresh()` for the initial state when the addon loads mid-proc
- `C_Timer.NewTicker` drives the border color interpolation (gray → gold over N ticks); cancelled on `GLOW_HIDE`
- Proc state is stored in `local procActive = false`; only the primary slot icon pulses (secondary slots do not pulse independently)

## Files Changed

- `HekiLight.lua` — proc-glow event handlers, border pulse ticker, IsSpellOverlayed check
