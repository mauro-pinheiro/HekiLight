# Story 8.1: Addon Icon & Minimap Button Branding

**Status:** InReview
**Epic:** 8 ‚Äî Addon Branding & UI Polish
**Version:** v0.6.1
**Priority:** Low ‚Äî cosmetic polish, no functional impact
**Complexity:** XS

## Context

HekiLight v0.6.0 shipped with three visual identity gaps:

1. **No AddOn list icon** ‚Äî the `?` placeholder appeared because `HekiLight.toc` had no `## IconTexture` directive.
2. **Generic minimap icon** ‚Äî the minimap button used `ability_whirlwind`, an unrelated spell icon with no connection to the addon.
3. **Blizzard tooltip warning** ‚Äî hovering the minimap button showed "This is a custom button. Please ask the addon author to use the standard LibDBIcon library instead." because the `OnEnter` script called `GameTooltip:SetOwner(self, ...)` on a `Minimap`-parented button. Blizzard hooks `GameTooltip` and intercepts for any non-LibDBIcon minimap child.

HekiLight intentionally does not use LibDBIcon (no LibStub dependency). The fix uses a private `GameTooltipTemplate` frame ‚Äî the same approach LibDBIcon itself uses internally ‚Äî to bypass Blizzard's intercept while keeping the button parented to `Minimap` (required for button-grouping addons such as LeatrixPlus).

## Acceptance Criteria

- [x] HekiLight shows its logo icon in the AddOn list (no `?` placeholder)
- [x] The minimap button displays the HekiLight logo instead of `ability_whirlwind`
- [ ] Hovering the minimap button shows "HekiLight / Click to open settings / Drag to reposition" with no Blizzard warning prepended
- [x] The minimap button remains grouped by LeatrixPlus (button stays parented to `Minimap`)
- [x] No regression to minimap button drag, click-to-open-settings, or visibility toggle

## Implementation Notes

- `assets/icon.tga` ‚Äî `logo.png` resized to 64√ó64 via `System.Drawing` (PowerShell); saved as PNG bytes with `.tga` extension, which WoW retail accepts
- `HekiLight.toc` ‚Äî added `## IconTexture: Interface\AddOns\HekiLight\assets\icon`
- `HekiLight.lua` `BuildMinimapButton()`:
  - `icon:SetTexture(...)` changed from `Interface\\Icons\\ability_whirlwind` to `Interface\\AddOns\\HekiLight\\assets\\icon`
  - Created `local tip = CreateFrame("GameTooltip", "HekiLightMinimapTooltip", UIParent, "GameTooltipTemplate")` inside `BuildMinimapButton`; `OnEnter`/`OnLeave` use `tip` instead of `GameTooltip`
