# Story 7.2: Cat Form Out-of-Combat Suggestion Display

**Status:** Done
**Epic:** 7 — Secondary Slot Active-Bar Filtering
**Version:** v0.6.0 (TBD)
**Priority:** High — Resto Druid + Wildstalker cannot pre-position on target before combat
**Complexity:** S (event registration + GetRotationSpells fallback + diagnostic cleanup)

## Context

In Cat Form with an attackable target out of combat, the suggestion display shows nothing. Entering combat immediately fixes it. `ShouldShow()` correctly returns true (via `UnitCanAttack("player","target")`), so display visibility is not the problem.

Root cause: `C_AssistedCombat.GetNextCastSpell(false)` returns nil for secondary-form spells out of combat. WoW's Rotation Assistant only pre-activates caster-form suggestions before combat; Cat Form suggestions for a Resto Druid require the poll loop (which only runs in combat). Without a primary spell, `GetSuggestionQueue` fills nothing and the display collapses.

Secondary contributing factor: `PLAYER_TARGET_CHANGED` is not registered. When the player selects a target after entering Cat Form, `ACTIONBAR_UPDATE_STATE` may not fire reliably (Cat Form action bar states may not change on target acquisition), so no Refresh is triggered.

Also in scope: remove the `DLog("SLOT2", ...)` diagnostic introduced during story 7.1 debugging, which fires at 20 Hz and floods the 500-entry session log buffer.

## Acceptance Criteria

- [x] In Cat Form with an attackable target out of combat, at least one suggestion icon appears
- [x] The suggestion shown is a valid Cat Form spell from the rotation queue
- [x] Entering Cat Form then selecting a target shows suggestions without requiring combat
- [x] No regression: caster form out-of-combat suggestions unaffected
- [x] No regression: combat suggestion flow unaffected (poll loop takes over as normal)
- [x] `DLog("SLOT2", ...)` removed from `GetSuggestionQueue` — buffer no longer flooded

## Implementation Notes

**Part 1 — Remove SLOT2 diagnostic flood:**
- Remove the 3-line `DLog("SLOT2", ...)` block from pass-1 of `GetSuggestionQueue` (~line 1449)

**Part 2 — Register `PLAYER_TARGET_CHANGED`:**
- Add `events:RegisterEvent("PLAYER_TARGET_CHANGED")` in the event registration block (~line 1963)
- In `OnEvent`, add `elseif event == "PLAYER_TARGET_CHANGED" then Refresh()` (same pattern as `ACTIONBAR_UPDATE_STATE`)
- This ensures Refresh fires reliably when a target is selected after shapeshifting in any form

**Part 3 — Primary slot fallback via `GetRotationSpells()`:**
- In `GetActiveSuggestion()`, when `GetNextCastSpell(false)` returns nil AND `not inCombat` AND `UnitCanAttack("player","target")`, attempt a pcall of `GetRotationSpells()` and iterate the result to find the first spell where `GetRealSlot(sid)` returns a non-nil slot AND `IsUsableAction(rslot)` is true (mirrors the story 7.1 filter)
- Use that spell + slot as the primary suggestion
- This gives Cat Form players the first usable rotation spell as a pre-combat suggestion

## Dependencies

- **Story 7.1 (InReview):** `IsUsableAction` guard must be in place — the Part 3 fallback inherits the same form-bar filter logic
- **Story 6.1 (Done):** Deferred keybind Refresh — keybinds will populate correctly via existing timer after form change

## Scope

**IN:** Remove SLOT2 DLog; register `PLAYER_TARGET_CHANGED`; `GetRotationSpells()` fallback in `GetActiveSuggestion` for out-of-combat nil case
**OUT:** No changes to `ShouldShow`, `StartPollLoop`, `StopPollLoop`, or the combat poll loop flow

## Business Value

Resto Druid + Wildstalker players entering Cat Form to DPS see what to cast immediately on target acquisition — without needing to auto-attack first to trigger the poll loop. Eliminates a jarring gap between target selection and first GCD suggestion.

## Risks

- **Low:** `GetRotationSpells()` out of combat may return a different ordering than in-combat. Acceptable — any valid Cat Form spell is better than blank.
- **Low:** `PLAYER_TARGET_CHANGED` fires on every target switch. `Refresh()` is already lightweight (called at 20 Hz in combat); occasional out-of-combat calls are negligible.
- **Low:** If `GetRotationSpells()` also returns nil out of combat in Cat Form, display remains blank — no regression, just no improvement for that edge case.
