# Story 7.1: Filter Secondary Slots to Current-Bar Spells Only

**Status:** Done
**Epic:** 7 — Secondary Slot Active-Bar Filtering
**Version:** v0.6.0 (TBD)
**Priority:** High — affects Restoration Druid + Wildstalker (common end-game build)
**Complexity:** XS (two-line guard in GetSuggestionQueue; no new data structures)

## Context

Restoration Druids with Wildstalker hero talents use Cat Form for melee/damage phases. `C_AssistedCombat.GetRotationSpells()` returns the full cross-form rotation queue, which includes Cat Form spells (e.g., Rip, Rake) even when the player is in caster or tree form.

`resolveSecondary(sid)` calls `GetRealSlot(sid)` → `FindSpellActionButtons(sid)`. The Cat Form bar occupies WoW action bar slots 73–84, and `FindSpellActionButtons` returns those slots even when not in Cat Form — so `rslot` is non-nil and a nil-check is insufficient.

`IsUsableAction(rslot)` returns `false` for form-restricted spells when the required form is not active. This is the correct WoW API to detect "spell blocked by form requirement."

## Acceptance Criteria

- [x] In caster/tree form, secondary slots show no Cat Form spells (Rip, Rake, etc.)
- [x] In Cat Form, secondary slots show Cat Form spells normally with correct keybinds
- [x] Balance Druid in Moonkin Form: Moonkin spells visible in secondary slots (no regression)
- [x] Balance Druid in caster form: Moonkin spells correctly absent from secondary slots
- [x] No regression for non-shapeshifting classes (Mage, Priest, Paladin, etc.)
- [x] `DLog("SLOT")` entries only appear for spells that passed the active-bar filter

## Implementation Notes

- In `GetSuggestionQueue` (~line 1452), both pass-1 and pass-2 conditions now include `and IsUsableAction(rslot)`:
  - Pass 1: `if effectiveID ~= primaryID and not IsSpellOnCooldown(sid) and rslot and IsUsableAction(rslot) then`
  - Pass 2: `if effectiveID ~= primaryID and IsSpellOnCooldown(sid) and rslot and IsUsableAction(rslot) then`
- Diagnostic `DLog("SLOT2", ...)` was added during debugging and floods the buffer at 20 Hz — removal is tracked in story 7.2

## Dependencies

- **EPIC-6 (Done):** Keybind cache robustness — `realSlotID` nil handling already present in Refresh render loop
- **HekiLight.lua:** `GetSuggestionQueue` (~line 1420), `resolveSecondary` (~line 1399)

## Scope

**IN:** `and rslot and IsUsableAction(rslot)` guard in pass-1 and pass-2 of `GetSuggestionQueue`
**OUT:** No changes to `resolveSecondary`, `GetRealSlot`, `GetActiveSuggestion`, `Refresh`, or event handlers

## Business Value

Eliminates visual noise for Restoration Druids with Wildstalker. Secondary slots show only actionable spells, making the suggestion display trustworthy regardless of current form.

## Risks

- **Low:** `FindSpellActionButtons` transient nil window after form changes (mitigated by EPIC-6 deferred timer). In the brief window, valid spells may also be filtered — acceptable, deferred Refresh at +0.3s restores them.
- **Low:** A player who manually places a form-specific spell on their main bar will still see it — correct behaviour, it is castable from that bar.
