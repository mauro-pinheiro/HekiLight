# Story 2.3: Keybind Labels on Secondary Slots

**Status:** Done
**Epic:** 2 — Multi-Suggestion Queue
**Version:** v0.1.3
**Commits:** `8743a99`, `8abc376`

## Context

The primary icon showed a keybind label, but the secondary strip icons did not. Players find it useful to see the keybind for upcoming spells — especially for planning ahead in multi-target or cooldown-heavy specs. The same taint-safe keybind pipeline used for the primary slot needed to be applied to each secondary slot.

## Acceptance Criteria

- [x] Each secondary slot icon displays a keybind label (same format as the primary)
- [x] Keybind lookup for secondary spells uses the same taint-safe `FindSpellActionButtons` path
- [x] All six main action bar pages are included in the slot→binding map
- [x] Missing keybinds degrade silently (label hidden, no error)

## Implementation Notes

- `RebuildSlotBindings()` already iterated pages 1–3; extended to pages 1–6 (`8abc376`) to cover all main bar pages
- Each `slots[i]` frame has its own `keybindText` FontString child, populated during `BuildSlots()` and updated in `Refresh()`
- The existing `GetSlotKeybind(spellID, realSlotID)` function is called once per secondary slot per refresh cycle; result cached in `keybindCache` (added in v0.3.0) for subsequent frames

## Files Changed

- `HekiLight.lua` — secondary slot keybind text, RebuildSlotBindings page range extension
