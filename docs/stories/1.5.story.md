# Story 1.5: Zero CPU Cost When RA Inactive + Assisted Highlight Support

**Status:** Done
**Epic:** 1 — Core Display
**Version:** v0.0.5
**Commits:** `ca87c92`, `3024ed1`, `1ece776`

## Context

The poll loop ran unconditionally at 20 Hz regardless of whether Rotation Assistant was configured or active. On characters without RA enabled, this burned CPU every 50ms for no reason. Additionally, the addon only detected the RA action bar button mode, missing the "Assisted Highlight" mode where Blizzard glows existing bar buttons instead of showing a dedicated RA slot.

## Acceptance Criteria

- [x] When RA is not configured/active, `OnUpdate` script is not set (zero CPU cost)
- [x] Poll loop starts immediately when RA becomes active (`IsAssistActive()` returns true)
- [x] Poll loop stops when RA becomes inactive
- [x] Both RA modes are detected: action bar button mode and Assisted Highlight mode
- [x] Icon remains visible during combat regardless of target state when RA is active

## Implementation Notes

- `IsAssistActive()` checks `C_ActionBar.HasAssistedCombatActionButtons()` — returns true for both RA modes
- `StartPollLoop()` / `StopPollLoop()` gate the `OnUpdate` script on `display`; the loop only runs when assist is active (see ADR-2)
- `GetActiveSuggestion()` two-layer detection: `C_AssistedCombat.GetNextCastSpell(false)` covers both modes; fallback via `FindAssistedCombatActionButtons()` handles any edge case (see ADR-3)
- `ACTIONBAR_SLOT_CHANGED` event restarts the loop when the RA button appears/disappears

## Files Changed

- `HekiLight.lua` — StartPollLoop/StopPollLoop gating, IsAssistActive(), Assisted Highlight detection
