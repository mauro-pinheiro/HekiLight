# Story 2.1: Multi-Suggestion Icon Strip

**Status:** Done
**Epic:** 2 — Multi-Suggestion Queue
**Version:** v0.1.0
**Commits:** `f09711c`, `92e8310`, `46f553d`

## Context

Experienced players want to see not just the next suggested spell but the next 2–3 spells in the rotation queue, so they can pre-position their fingers. Blizzard's `C_AssistedCombat.GetRotationSpells()` returns the full rotation queue. The addon needed to render secondary icon slots in a strip, respecting cooldown state so the player can see what's coming up even if it's not yet castable.

## Acceptance Criteria

- [x] Up to `db.maxSlots` secondary icon slots render in a horizontal strip below the primary icon
- [x] Each secondary slot shows the correct spell texture
- [x] Secondary slots are greyed out if the spell is on a real cooldown
- [x] Slots already shown as the primary suggestion are deduplicated from the strip
- [x] Secondary slot cooldown detection does not produce false-positives from the GCD (~1.5s)
- [x] Per-frame allocation from the queue is zero (pre-allocated `queueCache` table)

## Implementation Notes

- `GetSuggestionQueue()` calls `C_AssistedCombat.GetRotationSpells()` and populates `queueCache` in-place — no table allocation per tick (see ADR-10)
- Cooldown detection uses the pcall secret-number trick: wrap `duration == 0` in pcall; error → on real CD; success → available (see ADR-5). `GetSpellBaseCooldown` (>1500ms threshold) gates which spells enter `recentlyCastSpells`
- Per-cast cooldown tracking (`46f553d`): `UNIT_SPELLCAST_SUCCEEDED` adds spells to `recentlyCastSpells`; entries expire after `GetSpellBaseCooldown` ms
- Deduplication: `sid ~= primaryID` check in `GetSuggestionQueue` skips the spell already shown in the primary slot
