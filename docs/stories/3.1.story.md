# Story 3.1: Collapsible Settings Panel Consolidation

**Status:** Done
**Epic:** 3 — Settings & Polish
**Version:** v0.2.0 / v0.2.1
**Commits:** `0d5b2d8`, `cadc6b5`, `daadbf9`, `ea5c683`

## Context

As the settings panel grew (visibility toggles, icon size, scale, show mode, font customization, minimap angle, range alpha), it became a long vertical scroll with all sections expanded by default. The layout felt cluttered and overwhelming on first open. A collapsible section design was needed to keep the panel manageable.

## Acceptance Criteria

- [x] All settings are consolidated into a single panel with collapsible sections
- [x] All sections are collapsed by default on first open
- [x] Sections expand/collapse via a header button click
- [x] No horizontal scrollbar appears (content fits the canvas width)
- [x] Settings persist correctly after the layout change

## Implementation Notes

- `Settings.RegisterCanvasLayoutCategory` registers the single parent panel; sub-panels (Ignored Spells) use `RegisterCanvasLayoutSubcategory`
- Collapsible sections are `BackdropTemplate` frames with a header `FontString` and arrow indicator; `OnClick` toggles the child frame's `Show()`/`Hide()` and recalculates the layout scroll height
- Default collapsed state is enforced at `BuildSettingsPanel()` time — each section frame starts hidden
- Scrollbar auto-hide was attempted (`0d5b2d8`) but reverted (`cadc6b5`) because the Blizzard scroll frame API does not expose a stable auto-hide hook in Midnight 12.0
