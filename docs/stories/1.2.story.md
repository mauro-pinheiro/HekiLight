## Context

The icon alone does not tell the player which key to press. The addon needed to show the keybind for the suggested spell overlaid on the icon, matching the text WoW renders on action bar buttons. The challenge: Rotation Assistant slots are taint-protected — reading keybind data directly from them taints the action bar subsystem. A safe lookup path was required.

## Acceptance Criteria

- [x] The suggested spell's keybind is displayed as text overlaid on the icon (bottom-right corner)
- [x] The keybind reflects the actual spell binding, not the RA slot binding
- [x] Text uses THICKOUTLINE font so it is readable over any icon color
- [x] Falls back to the SBA slot binding if the spell has no direct binding on a regular bar
- [x] Works across all main action bar pages (page 1–6)

## Implementation Notes

- **Safe lookup path** (ADR-4): `spellID → C_ActionBar.FindSpellActionButtons(spellID) → filter RA slots via IsAssistedCombatAction → SLOT_BINDINGS[slot] → GetBindingKey`
- `SLOT_BINDINGS` is built once at `PLAYER_ENTERING_WORLD` by iterating all `ActionButton` frames and reading their `action` attribute
- `RebuildSlotBindings()` refreshes the map on `ACTIONBAR_SLOT_CHANGED` and `UPDATE_BINDINGS`
- `FormatKey()` strips the `BUTTON` prefix, abbreviates modifiers (`Ctrl-`, `Alt-`, `Shift-`)
- Fallback to SBA button binding handles specs that don't bind spells directly to bar slots
