# Story 6.1: Shapeshift-Resilient Keybind Cache Repopulation

**Status:** InReview
**Epic:** 6 — Keybind Robustness
**Version:** v0.5.0 (TBD)
**Priority:** High — affects all shapeshifting classes (Druid, Shaman ghost wolf, etc.)
**Complexity:** S (event handler tweak + one-shot timer; no new data structures)

## Context

When a Druid (or any shapeshifting class) exits Travel Form (or any non-caster form) and then acquires a target out of combat, the suggestion icon displays correctly but the keybind text is blank. It becomes visible only after entering combat.

**Root cause chain:**

1. Shapeshift fires `ACTIONBAR_SLOT_CHANGED`
2. Handler at ~line 1987 runs: `RebuildSlotBindings()` + `wipe(keybindCache)` + `Refresh()`
3. At that exact moment `FindSpellActionButtons` returns nil or RA-only slots (transient window after a slot change) — keybind lookup fails, cache stays empty
4. Target acquired → `ACTIONBAR_UPDATE_STATE` fires a **single** `Refresh()` call (out of combat, no poll loop)
5. Same transient miss — cache still empty — keybind renders `""`
6. Combat starts → `PLAYER_REGEN_DISABLED` starts the poll loop → repeated `Refresh()` calls eventually catch a non-transient moment → keybind resolves

The existing `keybindCache` fallback is the correct defence for the transient window, but it was just wiped, leaving nothing to fall back to.

## Acceptance Criteria

- [x] After shapeshifting out of a travel/feral/shapeshift form and acquiring a target out of combat, keybind text appears on the suggestion icon without requiring combat entry
- [x] The fix uses a one-shot deferred `Refresh()` (≥0.25 s) scheduled after `ACTIONBAR_SLOT_CHANGED`, giving `FindSpellActionButtons` time to settle
- [x] The deferred `Refresh()` is **cancelled/skipped** if combat starts before it fires (poll loop takes over)
- [x] A `DLog("KEYBIND", ...)` entry is emitted on each successful keybind resolution and on each cache-miss, tagged with spellID and source (`"slot"` vs `"cache"` vs `"miss"`)
- [ ] No regression on classes without shapeshift abilities
- [ ] `/hkl log 20` shows `KEYBIND` entries reflecting the resolution path used

## Implementation Notes

- Add a module-level `pendingKeybindRefresh` boolean or a lightweight timer handle; schedule a 0.3 s one-shot `C_Timer.After` in the `ACTIONBAR_SLOT_CHANGED` handler
- Guard: if `inCombat` is true when the timer fires, skip (`Refresh()` is already running via poll loop)
- `DLog("KEYBIND", ...)` should be added in `GetSpellKeybind`: one call on slot-resolved success, one on cache-hit fallback, one on full miss — tag with spellID and source
- Do **not** remove the `wipe(keybindCache)` on `ACTIONBAR_SLOT_CHANGED`; stale cache entries after a slot remap are worse than a brief blank. The deferred refresh repopulates it.
- `C_Timer.After` is safe out of combat; it does not taint

## Dependencies

- **EPIC-5 (Done):** No blocking stories. All correctness hardening complete.
- **HekiLight.lua:** Familiarity with `ACTIONBAR_SLOT_CHANGED` handler (~line 1987), `GetSpellKeybind`, and `keybindCache` required before implementation.
- **WoW API:** `C_Timer.After` — available since Legion, stable in Midnight 12.0.

## Scope

**IN:** Add `C_Timer.After(0.3, ...)` deferred refresh in `ACTIONBAR_SLOT_CHANGED` handler; add `KEYBIND` DLog tag in `GetSpellKeybind`; guard the timer against redundant combat-path firing.
**OUT:** No changes to `RebuildSlotBindings` logic. No changes to `keybindCache` wipe behaviour. No changes to the poll loop. No new events registered beyond what already exists.

## Business Value

Druid players (and any shapeshifter) see correct keybind text the moment they engage a target, matching the reliability already experienced by non-shapeshifting classes. Eliminates a visible correctness gap that could be mistaken for a broken addon.

## Risks

- **Low:** `C_Timer.After` callbacks are protected from taint. If the timer fires mid-combat (race condition), the `inCombat` guard prevents a redundant call.
- **Low:** Multiple rapid shapeshifts could queue multiple timers. Mitigate by cancelling any pending timer before scheduling a new one (store the handle, call `handle:Cancel()` if non-nil before `C_Timer.After`).
