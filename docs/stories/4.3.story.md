# Story 4.3: Poll Loop Lifetime Tied to Combat State

**Status:** Done
**Epic:** 4 ‚Äî Diagnostics & v0.3.0
**Version:** v0.3.2
**Commits:** `272c3fe`

## Context

After `ShouldShow()` was restored in v0.3.0, the poll loop continued running at 50ms between fights ‚Äî calling `GetNextCastSpell`, `GetRotationSpells`, and `GetSpellInfo` on every tick even with the display hidden and `ShouldShow()` returning false. This was pure wasted CPU. The loop needed to stop when combat ended and restart instantly when it began.

## Acceptance Criteria

- [x] The poll loop stops when combat ends (`PLAYER_REGEN_ENABLED`)
- [x] The poll loop restarts immediately when combat begins (`PLAYER_REGEN_DISABLED`) if RA is active
- [x] The first poll tick fires on the first frame after combat starts (no artificial delay)
- [x] Zero `OnUpdate` overhead between fights

## Implementation Notes

- `PLAYER_REGEN_ENABLED` handler calls `StopPollLoop()` before the final `Refresh()` (see ADR-8)
- `PLAYER_REGEN_DISABLED` handler calls `StartPollLoop()` if `IsAssistActive()` returns true
- `elapsed` accumulator in `StartPollLoop` is pre-seeded to `db.pollRate` so the first `OnUpdate` fires immediately without waiting one full poll interval
- `UNIT_HEALTH` event gated on `display:IsShown()` to avoid redundant visibility re-checks while hidden between fights

## Files Changed

- `HekiLight.lua` ‚Äî PLAYER_REGEN_ENABLED/DISABLED handlers, StopPollLoop on combat-end, UNIT_HEALTH visibility gate
