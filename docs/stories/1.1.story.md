# Story 1.1: Core Spell Suggestion Display

**Status:** Done
**Epic:** 1 — Core Display
**Version:** v0.0.1 (initial scaffold)
**Commits:** `a817b1a`, `5ff9630`, `0ff1355`, `345e64a`, `e434a0e`, `7aa85f9`

## Context

The Blood DK player wanted a movable, skinnable icon overlay that mirrors what Blizzard's Rotation Assistant (RA) suggests — similar to the old Hekili display. No such overlay existed; the RA button is embedded in the action bar and cannot be repositioned or resized. The addon needed to read Blizzard's suggestion and re-render it as a draggable frame.

## Acceptance Criteria

- [x] A movable icon frame appears when the Rotation Assistant has an active suggestion
- [x] The icon displays the correct spell texture for the suggested spell
- [x] The frame is visible at HIGH strata so it appears above the action bar
- [x] Position defaults to center-screen and persists across sessions via SavedVariables
- [x] `/reload` applies all Lua changes; no build step required
- [x] No Lua errors on load or during combat

## Implementation Notes

- `display` is the root `BackdropTemplate` frame; all child widgets are parented to it
- `GetActiveSuggestion()` uses two layers: primary `C_AssistedCombat.GetNextCastSpell(false)`, fallback via `FindAssistedCombatActionButtons()` + `GetActionInfo()` (both pcall-guarded)
- `GetActionCooldown` was replaced with `C_Spell.GetSpellCooldown` (Midnight 12.0 API returns a table, not three values)
- Frame strata set to `"HIGH"` with explicit frame level to render above Blizzard UI elements
- Dark semi-transparent background texture added for readability
