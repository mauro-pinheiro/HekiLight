# Story 2.2: Ignored Spells & Per-Character Storage

**Status:** Done
**Epic:** 2 — Multi-Suggestion Queue
**Version:** v0.1.0–v0.1.1
**Commits:** `dfb2798`, `62c9186`, `73a8404`, `d554a7d`, `525f326`, `24df7f4`, `c4c0a67`

## Context

The secondary slot strip showed maintenance spells (Revive Pet, Raise Ally, Raise Dead) alongside combat abilities, creating visual noise. Players needed a way to hide specific spells from the secondary list. Since ignored spells are class-specific (a Warlock's ignored list is meaningless on a Death Knight), the list had to be stored per-character, not account-wide.

## Acceptance Criteria

- [x] Players can ignore any spell from the secondary suggestion strip
- [x] Ignored spells are stored per-character (`HekiLightDBChar.ignoredSpells`)
- [x] Class-specific maintenance spells are auto-ignored on first load per character
- [x] The Ignored Spells list is accessible as a Settings sub-category
- [x] The dropdown shows only known spells (`IsPlayerSpell` filter)
- [x] Disabled (already-ignored) items in the dropdown are non-clickable
- [x] Slash commands can add/remove ignored spells

## Implementation Notes

- `HekiLightDB` (account-wide) holds display settings; `HekiLightDBChar` (per-character) holds `ignoredSpells` and `classDefaultsApplied` (see ADR-11)
- `CLASS_DEFAULT_IGNORED` table maps class name → list of spellIDs; applied once on first load via `classDefaultsApplied` flag
- `Settings.RegisterCanvasLayoutSubcategory` creates the Ignored Spells sub-panel under the main HekiLight category
- Rotation spell list for the dropdown is cached (`3d23df6`) to avoid calling `GetRotationSpells` on every dropdown open

## Files Changed

- `HekiLight.lua` — ignoredSpells logic, dropdown, CLASS_DEFAULT_IGNORED, per-character DB
- `HekiLight.toc` — `SavedVariablesPerCharacter: HekiLightDBChar`
