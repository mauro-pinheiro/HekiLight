# Story 1.3: UI Polish — Backdrop, Minimap Button, Settings Panel

**Status:** Done
**Epic:** 1 — Core Display
**Version:** v0.0.3
**Commits:** `8e90166`, `16b5167`, `547092b`

## Context

The raw icon frame needed finishing touches to feel like a polished addon: a styled border, a minimap button for easy toggle, a settings panel reachable from the game's Settings screen, a range pulse effect (icon dims when the suggested spell is out of range), and audio feedback.

## Acceptance Criteria

- [x] Backdrop border renders around the icon with configurable color
- [x] Minimap button appears at a configurable angle; drag to reposition
- [x] Settings panel is accessible from Game Menu → Interface → AddOns → HekiLight
- [x] Icon alpha pulses when the suggested spell is out of range
- [x] All layout controls (sliders, checkboxes) function correctly without flicker
- [x] Settings and position persist across sessions

## Implementation Notes

- Minimap button uses `math.atan2` to recalculate angle on drag; radius fixed at 80px from `Minimap:GetCenter()`
- Range pulse: `C_ActionBar.IsActionInRange(realSlotID)` (on a regular bar slot — see ADR-4) returns false → icon alpha set to `db.outOfRangeAlpha`
- `Settings.RegisterCanvasLayoutCategory` used for the panel; `OptionsSliderTemplate` is deprecated in Midnight — sliders built manually with `BackdropTemplate` and a `FontString` value label
- UI review pass (`16b5167`, `547092b`) fixed slider flicker caused by recursive `OnValueChanged` calls, corrected minimap icon anchor, and normalized backdrop insets
