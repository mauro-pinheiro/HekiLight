# Story 3.3: Defensive Floater (Built and Reverted)

**Status:** Reverted
**Epic:** 3 — Settings & Polish
**Version:** never shipped
**Commits:** `6730f8f` (add), `c18ab0c` (fix), `3b4a109` (taint fix attempt), `f898c67` (revert)

## Context

The player asked for a secondary floating frame that shows defensive cooldowns (Shield Wall, Icebound Fortitude, etc.) ready to use — similar to how some UIs display a dedicated "oh-shit button" tracker. `C_CooldownViewer` provides an "Essential" cooldowns list that includes Blizzard-curated defensive abilities. The feature was implemented and worked outside combat, but produced cascade taint errors during combat that blocked ability usage.

## Acceptance Criteria (original, not met)

- [ ] A secondary draggable frame shows defensive/essential cooldowns available to cast
- [ ] Shows only in combat
- [ ] Does not interfere with action bar inputs or cause taint errors

## Why It Was Reverted

`C_CooldownViewer` APIs touch Blizzard's protected frame internals when called from addon Lua during combat. The taint cannot be isolated with `pcall` — it propagates to the action bar and prevents the player from using abilities. Three fix attempts (`c18ab0c`, `3b4a109`) could not resolve the combat taint. The feature was reverted entirely (`f898c67`).

**Do not reintroduce** a defensive floater using `C_CooldownViewer`. Any future defensive tracking must use a different API path (e.g., tracking specific spell cooldowns via `GetSpellBaseCooldown` for known defensive IDs, entirely outside the `C_CooldownViewer` subsystem).
