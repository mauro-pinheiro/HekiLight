# Story 3.2: Keybind Font/Color Customization & "Always Show" Mode

**Status:** Done
**Epic:** 3 — Settings & Polish
**Version:** v0.1.6 → v0.1.9
**Commits:** `2731d4e`, `8abc376`, `a90f6f0`

## Context

Players with different UI scales, icon sizes, and personal preferences needed control over how the keybind text looks. Some run large icons where 10px font is unreadably small; others want specific colors for visibility against dark or bright backgrounds. Additionally, some players wanted the addon active at all times whenever RA had a suggestion — not just in combat — for use while preparing pulls or testing rotations on target dummies.

## Acceptance Criteria

- [x] Keybind font size is configurable (8–24px) via settings and `/hkl kbsize <N>`
- [x] Keybind text color is configurable (RGB 0–1) via settings and `/hkl kbcolor <r> <g> <b>`
- [x] Keybind outline style is configurable (outline / thick / none) via `/hkl kboutline`
- [x] Keybind text anchor corner is configurable (bottomright / bottomleft / topright / topleft / center) via `/hkl kbanchor`
- [x] A color picker widget is available in the settings panel for the keybind color
- [x] Show mode "always" makes the icon visible whenever RA has a suggestion, ignoring combat/target conditions

## Implementation Notes

- Font size and color changes call `keybindText:SetFont(...)` and `keybindText:SetTextColor(...)` on all slot frames immediately; no reload required
- `db.showMode = "always"` bypasses the `UnitCanAttack` and `InCombatLockdown` positive conditions in `ShouldShow()` while still respecting hard stops (dead/vehicle/cinematic)
- Color picker uses Blizzard's `ColorPickerFrame` (the standard game color picker dialog); `ColorPickerFrame.hasOpacity = false`
- All 6 main bar pages included in `RebuildSlotBindings()` slot scan to ensure keybinds are found regardless of which page the spell is bound to

## Files Changed

- `HekiLight.lua` — keybind font/color/outline/anchor customization, color picker, showMode="always"
