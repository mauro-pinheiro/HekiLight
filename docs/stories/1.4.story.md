# Story 1.4: Configurable Visibility Conditions

**Status:** Done
**Epic:** 1 — Core Display
**Version:** v0.0.4
**Commits:** `dd8629d`, `14f062e`, `2031994`, `5a89b44`, `befa077`, `fe9a29c`, `082aac8`

## Context

The icon should not appear in situations where following a rotation is irrelevant or disruptive — while dead, in a vehicle, during cinematics, while resting, or while mounted. But the right suppression rules are spec- and playstyle-dependent. Some players want the icon visible as long as RA has a suggestion; others only want it in combat with a hostile target. All conditions needed to be player-configurable.

## Acceptance Criteria

- [x] Icon suppresses when the player is dead or a ghost
- [x] Icon suppresses while in a vehicle or on a mount
- [x] Icon suppresses during in-game cinematics
- [x] Icon suppresses when resting (in an inn/city) — toggleable
- [x] Icon suppresses when no hostile/attackable target exists — toggleable
- [x] All hide conditions are individually toggleable via the Settings panel
- [x] All hide conditions are toggleable via `/hkl` slash commands
- [x] Being in combat overrides soft suppression conditions (rest, target)
- [x] An attackable target overrides all soft suppression conditions

## Implementation Notes

- v0.1 used hide conditions (suppress-if-any-true); `a4c9c1a` refactored to positive show conditions (show-if-any-true); `48e6914`/`082aac8` restored hard stops separated from soft conditions — the two-tier `ShouldShow()` that remains today (see ADR-7)
- Hard stops: `UnitIsDeadOrGhost("player")`, `UnitInVehicle("player")`, cinematic flag
- Soft conditions: `UnitCanAttack("player","target")`, `InCombatLockdown()`
- `ALWAYS_HIDE_FLAGS` and `SHOW_FLAGS` tables drive bulk slash command toggles; `db.showMode = "always"/"active"` is the user-facing setting
